# GitHub 仓库设置指南

完成以下设置让你的开源项目更专业。

## 1. 仓库基本信息

访问: https://github.com/Yonsun-w/multi-repo-dev/settings

### About 部分（右侧边栏）

**Description** (中文):
```
Claude Code 多仓库协同开发工作流 Skill - 从需求到代码的全流程自动化
```

**Description** (英文):
```
Multi-repository development workflow skill for Claude Code - Full automation from requirements to code
```

**Website**:
```
https://github.com/Yonsun-w/multi-repo-dev#readme
```

**Topics** (标签):
添加以下 topics：
- `claude-code`
- `claude-skill`
- `workflow`
- `multi-repo`
- `multi-repository`
- `development-tools`
- `ai-assistant`
- `git-workflow`
- `requirement-management`
- `code-generation`

## 2. 创建 GitHub Release

访问: https://github.com/Yonsun-w/multi-repo-dev/releases/new?tag=v1.0.0

按照 `CREATE_RELEASE_GUIDE.md` 中的步骤创建 Release。

## 3. 启用功能

在 Settings 页面：

### General
- ✅ Issues (已启用)
- ✅ Discussions (启用讨论功能)

### Features
- ✅ Wikis (可选，用于更详细的文档)
- ✅ Projects (可选，用于项目管理)

## 4. 设置社交预览图（可选）

在 Settings → Social preview 上传一张 1280x640px 的预览图。

可以使用在线工具生成：
- https://og-image.vercel.app/
- https://www.bannerbear.com/

## 5. 添加 .github 目录（可选）

创建以下文件增强项目专业度：

### Issue 模板
`.github/ISSUE_TEMPLATE/bug_report.md`
`.github/ISSUE_TEMPLATE/feature_request.md`

### Pull Request 模板
`.github/pull_request_template.md`

### GitHub Actions（可选）
`.github/workflows/ci.yml` - 自动化测试

## 快速操作链接

- **仓库设置**: https://github.com/Yonsun-w/multi-repo-dev/settings
- **创建 Release**: https://github.com/Yonsun-w/multi-repo-dev/releases/new?tag=v1.0.0
- **Topics 设置**: 点击仓库首页右侧 About 旁边的齿轮图标
- **启用 Discussions**: Settings → General → Features → Discussions
